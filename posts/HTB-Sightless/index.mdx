The nmap result doesn't show any special ports open so it seems as we will need to exploit the web application for initial foothold. To access the web application add `sightless.htb` to `/etc/hosts` file.

```bash
##
# Host Database

# localhost is used to configure the loopback interface
# when the system is booting.  Do not change this entry.
##
127.0.0.1	localhost
255.255.255.255	broadcasthost
::1             localhost
10.129.199.225	sightless.htb
```

The web application has a interesting subdomain on the our services section of the page. The subdomain is the following `sqlpad.sightless.htb`.

We will need to add `sqlpad.sightless.htb` to our `/etc/hosts` file to access the subdomain web application. Here's a overview of my `/etc/hosts` file.

We successfully managed to acces the `sqlpad.sightless.htb` application. The version of SQLPad the web application is running is `6.10.0` which is a interesting information.

<img src="/assets/images/HTB-Sightless/HTB-Sightless-2.png" />

According to multiple of sources the SQLPad version 6.10.0 is vulnerable for remote code execution. I decided to use the following exploit on the machine [SQLPad-RCE-Exploit-CVE-2022-0944](https://github.com/0xRoqeeb/sqlpad-rce-exploit-CVE-2022-0944) to potentially get a remote code execution on the machine.

<img src="/assets/images/HTB-Sightless/HTB-Sightless-3.png" />

Success! We finally have a remote code execution on the machine. However, it seems as we are inside a docker container as we are logged in as root instead of www-data or a user.
